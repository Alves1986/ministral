## TypeScript
- **strict mode** obrigatório (`tsconfig.json`)
- Proibido `any` — usar `unknown` quando necessário
- Tipos globais centralizados em `types.ts`

## React
- Apenas **functional components** com hooks
- **Proibido** class components
- Estado de servidor via **TanStack Query** (não `useEffect + fetch`)
- Estado de UI global via **Zustand** (`store/appStore.ts`)
- Estado de sessão via **React Context** (`context/SessionContext.tsx`)

## Supabase
- Client **singleton**: `services/supabase/client.ts` — nunca criar nova instância
- Toda query DEVE filtrar por `organizationId` + `ministryId` (RLS)
- Queries sem filtro de org/ministry = bug crítico

## Naming conventions
- Componentes: `PascalCase.tsx` (ex: `AvailabilityScreen.tsx`)
- Serviços: `camelCase.ts` (ex: `scheduleServiceV2.ts`)
- Stores: `camelCase.ts` com sufixo `Store` (ex: `appStore.ts`)
- Edge Functions: `kebab-case/` (ex: `whatsapp-reminders/`)
